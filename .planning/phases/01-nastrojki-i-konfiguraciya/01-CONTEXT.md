# Phase 1: Настройки и конфигурация - Context

**Gathered:** 2026-05-09
**Status:** Ready for planning

<domain>
## Phase Boundary

Пользователь может задать пути к файлам и параметры систем через боковую панель настроек, и эти значения сохраняются между запусками приложения. Фаза добавляет панель настроек к существующему UI-прототипу.

Входит в скоуп: открытие/закрытие панели настроек, три поля с путями, сохранение в JSON, валидация путей.
Не входит: интеграция с реальными системами, запуск процессов, история запусков.

</domain>

<decisions>
## Implementation Decisions

### Открытие и отображение панели

- **D-01:** Панель настроек открывается кнопкой-иконкой ⚙ в правом углу хедера основного окна (позиция ~1080px, рядом с текущим статусом "All services online").
- **D-02:** Панель — боковая (не отдельное окно Form), выезжает справа поверх карточек этапов. Ширина панели ~350px.
- **D-03:** Появление/скрытие — анимированное скольжение справа, реализованное через System.Windows.Forms.Timer (паттерн уже есть в прототипе — CardTimer для карточек этапов). Анимация закрытия — обратная.
- **D-04:** Кнопки в футере панели: «Сохранить» + «Отмена». «Отмена» закрывает панель без сохранения.

### Поля настроек (Phase 1)

- **D-05:** В Phase 1 три поля:
  1. **Путь к .spp файлу** (Plant Simulation) — обязательное поле, OpenFileDialog с фильтром `*.spp`
  2. **Рабочий каталог** — папка для временных файлов/логов, FolderBrowserDialog
  3. **Папка со скриптами/макросами** — папка где лежат .lsp, .mac и другие файлы интеграций, FolderBrowserDialog
- **D-06:** Каждое поле — TextBox (только чтение) + кнопка «…» справа. Кнопка «…» открывает соответствующий диалог (OpenFileDialog или FolderBrowserDialog).

### Хранение настроек

- **D-07:** Настройки хранятся в `settings.json` в той же папке, что и .exe ($PSScriptRoot). Формат: плоский JSON-объект с ключами `PlantSimPath`, `WorkDir`, `ScriptsDir`.
- **D-08:** Загрузка при старте: если `settings.json` существует — ConvertFrom-Json → заполнить TextBox'ы. Если не существует — поля пустые.
- **D-09:** Сохранение: ConvertTo-Json → Out-File с -Encoding UTF8.

### Валидация

- **D-10:** Валидация срабатывает при нажатии «Сохранить» (не on-blur, не real-time).
- **D-11:** Ошибка несуществующего пути: красный бордер вокруг TextBox + мелкая подпись под полем (например, «Путь не найден»). Использовать цвета по аналогии с $colSuccessBg/Text — ввести $colErrorBg и $colErrorText.
- **D-12:** Если все пути валидны (или пусты — пустые поля допустимы для незаполненных полей) → JSON сохраняется, панель закрывается.

### Claude's Discretion

- Точные размеры шрифтов и отступы в панели — следовать существующей палитре прототипа
- Порядок полей в панели
- Заголовок панели настроек (например, «Настройки» или «Конфигурация»)
- Placeholder-текст в пустых TextBox'ах

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- **Цветовая палитра** (`$colBgPage`, `$colBgCard`, `$colBorder`, `$colInfoBg`, `$colInfoText`, `$colSuccessBg`, `$colSuccessText`, `$colTextPrimary`, `$colTextSec` и др.) — использовать напрямую, не вводить новые цвета без необходимости.
- **Timer-анимация** (CardTimer, паттерн на строках 231-278) — переиспользовать для анимации боковой панели (изменение Width вместо изменения Size карточки).
- **FlatStyle кнопки** (`FlatStyle = "Flat"`, `FlatAppearance.BorderColor = $colBorder`) — стиль кнопок «Сохранить»/«Отмена» и «…».
- **Add_Paint для кастомного бордера** — паттерн для отрисовки рамки панели настроек.

### Established Patterns
- **Все контролы добавляются в `$card`** (основная Panel), а не в `$form` напрямую.
- **`Add_Paint`** используется для кастомной отрисовки (рамки, иконки) — не BackColor для цветных элементов.
- **`New-Object System.Drawing.Font("Segoe UI", N)`** — шрифт везде одинаковый.
- **Фиксированный размер окна** (1200×620, FixedSingle) — боковая панель перекрывает содержимое, а не расширяет окно.

### Integration Points
- Иконка ⚙ добавляется в `$card` с позицией ~(1080, 22) — рядом с `$statusPanel` (960, 28).
- Боковая панель (`$settingsPanel`) добавляется в `$card`, BringToFront() при открытии.
- При нажатии «Сохранить» — записывает `settings.json`, закрывает панель анимацией.
- При старте `$form` (`$form.Add_Load` или перед `ShowDialog`) — читает `settings.json` и заполняет глобальные переменные настроек.

</code_context>

<specifics>
## Specific Ideas

- Анимация боковой панели аналогична CardTimer в прототипе: Timer.Interval=12, шаг изменения ширины по 8-12px, от 0 до ~350px (выезд) и обратно (закрытие).
- Иконка ⚙ — через Add_Paint с DrawString символа ⚙ ([char]0x2699) в стиле существующей headerIcon.
- Валидация: `Test-Path $textBox.Text` — если false и поле не пустое → красный бордер.

</specifics>
